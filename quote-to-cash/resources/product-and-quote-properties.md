# Product and Quote Properties Reference

Complete property reference for the four quote-to-cash object types.

---

## Products (`--type products`)

| Property | Type | Required | Notes |
|---|---|---|---|
| name | string | Yes | Display name in the product catalog |
| description | string | No | Product description |
| price | number | No | Unit price in the portal's default currency |
| hs_sku | string | No | Stock-keeping unit identifier |
| hs_cost_of_goods_sold | number | No | COGS ‚Äî used for margin calculations |
| recurringbillingfrequency | enumeration | No | See enum values below |
| hs_product_type | string | No | Freeform product type label |

**recurringbillingfrequency enum values:**

| Value | Description |
|---|---|
| `monthly` | Billed monthly |
| `quarterly` | Billed every 3 months |
| `per_six_months` | Billed every 6 months |
| `annually` | Billed once per year |
| `per_two_years` | Billed every 2 years |
| `per_three_years` | Billed every 3 years |

Leave `recurringbillingfrequency` unset for one-time purchases.

---

## Line Items (`--type line_items`)

Line items are the individual entries on a quote. They can be standalone or linked to a product via `hs_product_id`.

| Property | Type | Required | Notes |
|---|---|---|---|
| name | string | Yes | Display name on the quote |
| quantity | number | Yes | Number of units |
| price | number | No | Unit price ‚Äî overrides product price if set |
| hs_product_id | string | No | Links this line item to a Product record |
| discount | number | No | Discount percentage (e.g., `10` = 10% off) |
| hs_total_discount | number | No | Read-only ‚Äî computed by HubSpot from `discount` √ó `price` √ó `quantity` |
| hs_line_item_currency_code | string | No | ISO 4217 currency code, e.g., `USD`, `EUR`, `GBP` |
| hs_position_on_quote | number | No | Display order integer ‚Äî lower numbers appear first |

**Do not set `hs_total_discount` directly.** Set `discount` (percentage) and HubSpot computes the total.

## Quotes (`--type quotes`)

Quotes are always created as `DRAFT` from the CLI. Status transitions to `APPROVAL_NOT_NEEDED` or `PENDING_APPROVAL` are required before sharing with a customer.

| Property | Type | Required | Notes |
|---|---|---|---|
| hs_title | string | Yes | Quote title displayed to the customer |
| hs_expiration_date | string | No | `YYYY-MM-DD` format ‚Äî date the quote expires |
| hs_status | enumeration | No | See enum values below |
| hs_currency | string | No | ISO 4217 currency code, e.g., `USD` |
| hs_public_url_key | string | No | Read-only ‚Äî generated by HubSpot, used in the shareable URL |
| hs_pdf_download_link | string | No | Read-only ‚Äî direct link to download the quote as a PDF |

**hs_status enum values:**

| Value | Description |
|---|---|
| `DRAFT` | Default ‚Äî editable, not visible to the customer |
| `APPROVAL_NOT_NEEDED` | Ready to send ‚Äî no approval workflow configured |
| `PENDING_APPROVAL` | Waiting for internal sign-off |
| `APPROVED` | Approved and can be shared with the customer |
| `REJECTED` | Approval was denied |
| `ARCHIVED` | Soft-deleted / no longer active |

**Workflow note:** The CLI can create quotes and update status, but the shareable link and PDF generation are only accessible after the quote is approved and shared via the HubSpot UI.

## Invoices (`--type invoices`)

The CLI can read and update invoice status. Creating invoices from scratch requires HubSpot Commerce to be configured on the portal.

| Property | Type | Required | Notes |
|---|---|---|---|
| hs_invoice_status | enumeration | No | See enum values below |
| hs_due_date | string | No | `YYYY-MM-DD` format |
| hs_number | string | No | Invoice number (often auto-generated) |
| hs_amount_billed | number | No | Total invoice amount |
| hs_balance | number | No | Remaining unpaid balance |
| hs_currency_code | string | No | ISO 4217 currency code |

**hs_invoice_status enum values:**

| Value | Description |
|---|---|
| `DRAFT` | Not yet sent |
| `OUTSTANDING` | Sent, payment not yet received |
| `PAID` | Payment received in full |
| `VOIDED` | Cancelled ‚Äî no payment expected |
| `OVERDUE` | Past due date with unpaid balance |

## Association Requirements

After creating line items and quotes, associations are required to connect them:

| From | To | Purpose |
|---|---|---|
| `quotes` | `line_items` | Attach line items to the quote |
| `deals` | `quotes` | Link the quote to the deal for CRM visibility |
| `quotes` | `contacts` | Optional ‚Äî associate customer contact to quote |

```bash
hubspot associations create --from quotes:<quote_id> --to line_items:<line_item_id>
hubspot associations create --from deals:<deal_id> --to quotes:<quote_id>
```

Multiple line items: create each line item separately and associate each to the same quote.
